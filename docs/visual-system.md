# Habit Flywheel Visual System

> 适用于 `Welcome / Auth / App / Dialogs` 的统一视觉方案。
> 目标不是“小修一下颜色”，而是把产品从“紫金积分工具感”重构为“欲望验证的个人系统”。

## 1. 视觉判断

当前 UI 的主要问题不在布局，而在配色方向：

- 全局仍受旧版 `紫 + 金 + 黑` 主导。
- 欢迎页和应用内大量复用紫色发光、紫金渐变和高饱和强调。
- 弹窗层与页面层没有拉开语义差异，`创建 / 查看 / 兑现 / 放弃` 看起来像同一种表层组件。

这会导致三个结果：

1. 视觉气质更像“通用 AI SaaS 模板”，而不是有方法论的产品。
2. 奖励系统更像游戏积分，而不是“欲望验证”。
3. 关键状态和关键动作虽然功能不同，但用户体感上不够分层。

## 2. 新方向

建议将整体视觉方向统一为：

`纸白 / 墨绿 / 铜色`

关键词：

- 克制
- 有仪式感
- 带产品叙事感
- 不幼稚游戏化
- 不做紫色默认 AI 风

一句话定义：

> 不是“奖励型习惯 App”，而是“验证欲望并兑现行动的个人操作系统”。

## 3. 核心配色原则

### 3.1 色彩结构

- 主背景：暖纸白或浅矿物灰，不用纯白。
- 主文字：深墨色，不用纯黑。
- 主色：深青绿 / 墨绿，承担“执行、推进、可信赖”。
- 强调色：铜色 / 琥珀棕，承担“奖励、里程碑、兑现”。
- 辅助色：灰褐、雾蓝、苔绿，用于层级和低强度反馈。

### 3.2 颜色职责

- 不再让两个高饱和主色同时竞争。
- 页面主结构用低饱和中性色和深青绿建立秩序。
- 只有奖励、阈值、兑现、关键 CTA 才使用铜色系。
- 危险动作统一走低饱和铁锈红，而不是泛用亮红。

### 3.3 推荐 Token 方向

#### 浅色模式

- `--background`: 暖纸白
- `--card`: 米白
- `--foreground`: 深墨灰
- `--primary`: 深青绿
- `--primary-foreground`: 浅纸白
- `--accent`: 铜色
- `--muted`: 浅灰米色
- `--border`: 暖灰边框

#### 深色模式

- `--background`: 炭黑偏绿
- `--card`: 深石墨
- `--foreground`: 暖白
- `--primary`: 浅青绿
- `--accent`: 暖铜色
- `--muted`: 深灰褐
- `--border`: 低对比石墨边框

## 4. 状态颜色语义

计划状态不能只换文案，应该有稳定的颜色语义：

- `wish`
  - 中性灰
  - 表示尚未投入，仅是待验证欲望
- `validating`
  - 铜棕 / 沙金
  - 表示观察期、验证期、试探投入
- `active`
  - 深青绿
  - 表示已确认值得追，进入稳定执行
- `ready_to_redeem`
  - 更亮的铜色
  - 表示已到阈值，等待结算
- `redeemed`
  - 苔绿 / 深绿色
  - 表示完成闭环
- `abandoned`
  - 灰红 / 铁锈红
  - 表示明确放弃，但不是灾难性失败

## 5. 弹窗层配色

有，弹窗页面必须单独考虑，而且应该比普通页面更严格。

原因：

- 弹窗不是普通卡片，而是“高意图操作层”。
- 在 Habit Flywheel 里，弹窗经常承载状态切换、兑现确认、放弃反思等高语义动作。
- 如果弹窗继续沿用普通卡片颜色，只会显得“浮在页面上”，不会形成仪式感。

### 5.1 通用弹窗原则

- 遮罩层：不要纯黑蒙层，改成带一点墨绿或暖灰倾向的半透明遮罩。
- 弹窗底色：比普通卡片更稳、更实，不要继续发紫。
- 边框：低对比边框，局部使用状态色高亮。
- 标题区：可以有轻微的顶部色带或图标强调，但不要整块高饱和渐变。
- 主按钮：始终只允许一个强主色。
- 次按钮：回到中性色，不和主按钮抢权重。

### 5.2 按语义区分弹窗

#### `PlanComposer`

- 气质：构建、规划、开始一轮新验证
- 颜色：中性底 + 深青绿主按钮 + 轻铜色步骤强调
- 不要：整页紫色渐变

#### `PlanDetail`

- 气质：运营面板
- 颜色：中性底 + 状态色作为局部标签和按钮强调
- 不要：把详情弹窗做成“高光卡”

#### `RedeemPlanDialog`

- 气质：兑现仪式
- 颜色：铜色高亮最强，但只集中在标题图标、关键数字、确认按钮、细节描边
- 背景仍应克制，避免做成“领奖页”

#### `AbandonPlanDialog`

- 气质：克制的终止与反思
- 颜色：灰红 / 铁锈红点到为止，背景仍然中性
- 要求：更安静、更严肃，不要强刺激红黑对撞

#### `LegacyWorkspaceDialog`

- 气质：兼容层 / 旧系统入口
- 颜色：去饱和、偏灰、弱强调
- 目的：让用户感受到“它还可用，但不是主舞台”

## 6. 应用内页面建议

### 6.1 App Shell

- 侧边栏不再使用紫金 Logo 球感。
- 可改为墨绿底上的铜色小强调，或米白底 + 深墨绿 icon。
- Hero 区不要再使用紫金渐变大面积铺底，改为：
  - 纸白底
  - 局部深青绿块面
  - 小范围铜色提示

### 6.2 Dashboard

- 当前主计划：深青绿主导
- 兑现距离：铜色数字强调
- Validation Signals：用低饱和语义色区分，而不是统一一种 accent

### 6.3 Review

- 趋势图可以保留双线，但改成：
  - 深青绿：行动/完成
  - 铜色：能量/兑现进度
- 卡片背景尽量安静，不要继续彩色铺底

### 6.4 Library / Settings / Compatibility

- 这三页不该再用“主舞台色”
- 应更多使用中性背景与低饱和边框
- 让用户体感到它们是支撑层，而不是执行层

## 7. 欢迎页改造方向

欢迎页的问题比应用内更严重，因为它现在还带有典型的“紫色产品落地页”气质。

建议改成：

`编辑感宣言页 + 纸面系统感`

### 7.1 Hero

- 去掉顶部紫色 radial glow
- 改成暖纸白背景 + 大留白 + 极浅青灰/铜色几何块
- 右侧预览卡片改成“产品样本页”，而不是发光 UI demo

### 7.2 CTA

- 主按钮：深青绿实底
- 次按钮：铜色描边或中性色描边
- 不再使用紫色主 CTA

### 7.3 内容区

- 功能卡片不要都用紫色 hover
- 不同 section 用：
  - 纸白
  - 浅石灰
  - 极浅青灰
  形成层级

### 7.4 叙事卡片

- 引言、流程、故事区应该更像产品手册，而不是功能营销页
- 可用铜色细线、编号圆点、纸页边框、轻纹理背景来增强识别

## 8. Typography 建议

当前如果只换颜色，不换字体气质，效果会有限。

建议：

- 欢迎页标题可使用更有编辑感的 serif 或 quasi-serif
- 应用内保留清晰无衬线，但避免太普通的默认系统感
- 标题字重和正文灰度要拉开，不要靠高饱和颜色补层级

## 9. 实施顺序

### Step 1

先重构全局 Token：

- `src/index.css`
- 统一 light / dark 的背景、卡片、主色、accent、状态色

### Step 2

再重构认证后壳层和核心卡片：

- `src/pages/Index.tsx`
- `src/components/vnext/*`

### Step 3

再处理所有高语义弹窗：

- `PlanComposer`
- `PlanDetailDialog`
- `RedeemPlanDialog`
- `AbandonPlanDialog`
- `LegacyWorkspaceDialog`

### Step 4

最后重做欢迎页：

- `src/pages/Welcome.tsx`

## 10. 完成标准

- 不再出现大面积紫金渐变主导的页面
- 页面层、卡片层、弹窗层形成明确视觉层级
- `验证中 / 进行中 / 可兑现 / 已兑现 / 已放弃` 状态在颜色上可一眼区分
- 欢迎页和应用内共享同一气质，但各自承担不同叙事角色
- 弹窗不再只是浮层，而是具有操作语义和仪式感的高意图界面
